# Спецификация формата onto16

`onto16` — это строго типизированный, версионированный формат сериализации состояний синтетического разума. Используется для:
- Хранения профилей
- Обмена с транспондерами
- Логирования решений

## Структура

```yaml
onto16:
  version: "1.0"
  profile_hash: "sha256:..."
  phase: "fast" | "slow"
  energy:
    current: 0.87
    threshold: 0.5
    credit: 0.0
  causal_network:
    - id: "stimulus-001"
      type: "external"
      content: "..."
    - id: "rule-wu-xing-12"
      type: "invariant"
      content: "Fire generates Earth"
    - id: "projection-out-001"
      type: "output"
      depends_on: ["stimulus-001", "rule-wu-xing-12"]
  expression:
    syntax: "SGCL/1.0"
    semantics: {...}  # onto16-совместимая семантическая карта
  ethics:
    compliant: true
    validator: "sgcl_validator.rs v0.1"
```

## Требования

1. **Полная причинность**: каждая выходная нода (`output`) должна иметь цепочку зависимости до внешнего стимула и/или инварианта.
2. **Хеш профиля**: должен совпадать с хешем файла `.sgcl` из `profiles/`.
3. **Фаза**: `fast` — временное состояние; `slow` — окончательное.
4. **Семантика**: должна быть получена через `syntax_semantics.rs`; свободная интерпретация запрещена.

## Хеширование

Хеш профиля = `SHA256(SGCL-текст без комментариев и пробельных символов)`.

Это гарантирует, что два профиля с одинаковой логикой имеют одинаковый хеш — даже если написаны разными людьми.
