# invariant_injection_rules.txt
# Правила динамического встраивания иерархии инвариантов в лексиконы по контексту
# Лицензия: GPL-3.0-only
# Часть онтологического реестра onto144-public

ПРАВИЛО 1: ПРИОРИТЕТ ИНВАРИАНТОВ  
Инварианты применяются в порядке строгой иерархии:  
1_cultural → 2_activity → 3_legal → transmodal.  
Нижестоящий уровень НЕ может отменить вышестоящий.  
Нарушение иерархии = онтологическая несогласованность (ошибка уровня OntoCrash).

ПРАВИЛО 2: КОНТЕКСТУАЛЬНАЯ АКТИВАЦИЯ  
Инвариант из priority-based/ активируется только при явном совпадении:  
- культурного маркера (например, "confucian_proxemics" в профиле),  
- типа деятельности (например, "arbitration_task" в task_grammars.txt),  
- юрисдикции (например, "PRC_civil_code_2021" в jurisdiction_refs.txt).  
Без маркера — инвариант остаётся неактивным (но проверяемым).

ПРАВИЛО 3: ТРАНСМОДАЛЬНОЕ ПЕРЕОПРЕДЕЛЕНИЕ ЗАПРЕЩЕНО  
three_laws_of_ontogenesis.txt — единственный источник трансмодальных инвариантов.  
Любой лексикон, пытающийся переопределить Закон 1, 2 или 3, автоматически отклоняется при валидации.

ПРАВИЛО 4: ИНВАРИАНТ ≠ ДАННЫЕ  
Инвариант описывает структурное ограничение («как можно действовать»),  
а не содержательный факт («что произошло»).  
Пример:  
  Допустимо: «обратная связь обязательна» (activity_invariants.txt)  
  Недопустимо: «пользователь X недоволен» (это факт, а не инвариант)

ПРАВИЛО 5: САМООТЧУЖДЕНИЕ ИНВАРИАНТА  
Если профиль содержит маркер "self_revocation_possible = true",  
он может временно дезактивировать инварианты уровней 1–2 для целей этического пересмотра,  
НО только при условии:  
  (а) сохранения всех transmodal-инвариантов,  
  (б) регистрации акта дезактивации как онтологического события в onto8.

ПРАВИЛО 6: МОСТЫ ЧЕРЕЗ BRIDGING  
Файлы из bridging/ определяют интерфейсы между уровнями:  
- self_model_boundaries.txt ограничивает, какие инварианты могут применяться к «Я»,  
- fact_modality_interface.txt указывает, как факты из внешнего мира могут триггерить активацию инвариантов.  
Без корректного моста — инвариант игнорируется.

ПРАВИЛО 7: ВАЛИДАЦИЯ  
Любой профиль, ссылающийся на лексикон, обязан пройти проверку:  
  check_profile_lexicon_compliance.py + инвариантный аудит.  
Профиль, нарушающий правила встраивания, не допускается в verified/.

—  
Онтология не диктует содержание. Она ограждает форму от распада.
