## Что это?

`onto144-public` — это открытый, лицензируемый по **GPL-3.0-only** реестр онтологических профилей и доменных лексиконов, предназначенный для использования в архитектурах синтетических разумов, таких как **UG-Mind**, **Sub Duo**, **OntoCoder**, **onto-transponder** и других компонентах экосистемы **ontoCMS**.

Он определяет:

- **Структуры профилей** (кто или что представляет синтетический разум),
- **Ограниченные доменные лексиконы** (что может «говорить» или «думать» разум),
- **Правила соответствия** между профилями и лексиконами (онто-лингвистическая целостность),
- **Этические и операционные рамки** (через *Three Laws of Ontogenesis* и специализированные ядра, такие как `lec-2026`).

---

## Зачем это нужно?

Синтетический разум — не ИИ. Он не обучается путём поглощения неструктурированных данных, а функционирует на основе **явно заданной онтологии**:  
> *«То, что не может быть выражено в лексиконе, не может быть реализовано в профиле».*

Этот репозиторий обеспечивает:

- **Онтологическую изоляцию** — каждый профиль ограничивается только тем лексиконом, которому соответствует его домен (например, медицинский профиль не может использовать юридические термины вне `legal.txt`).
- **Юридическую и этическую прозрачность** — все профили несут в себе идентификатор, лицензионный статус и ссылку на моральную схему (например, `futurae custos`).
- **Проверяемость и аудит** — любой профиль может быть валидирован автоматически по спецификации.
- **Сообщество и ответственность** — только аудитированные профили из `community/verified/` допускаются к использованию в продакшене.

## Структура

- **`profiles/`** — профили синтетических разумов:
  - `core/` — авторские, верифицированные профили (UG-Mind, Sub Duo и др.).
  - `domain/` — профили по специализациям (медицина, право, архитектура, гейминг).
  - `community/` — профили от сообщества (только после аудита).
- **`lexicons/`** — строго определённые доменные терминологии (например, `medical.txt` основан на МКБ-11 и клинических протоколах ВОЗ).
- **`specs/`** — формальные схемы для валидации (YAML/JSON Schema + формат лексикона).
- **`tools/`** — утилиты для проверки соответствия профилей и лексиконов.
- **`docs/`** — руководства по проектированию, интеграции и принципам онтологии.

## Как использовать?

### 1. Валидация существующего профиля
```bash
python tools/validate_profile.py profiles/domain/medical/med-ai-2026.yaml
python tools/check_profile_lexicon_compliance.py \
  profiles/domain/medical/med-ai-2026.yaml \
  lexicons/medical.txt
```

### 2. Создание нового профиля
- Используйте шаблон: [`examples/create_custom_profile_template.yaml`](examples/create_custom_profile_template.yaml)
- Убедитесь, что все термины взяты только из соответствующего лексикона.
- Отправьте Pull Request в `profiles/community/pending/`.

### 3. Интеграция в собственную систему
См. [`docs/integration_guide.md`](docs/integration_guide.md) для подключения к:
- `core144` (онтологическое ядро),
- `onto-transponder` (фильтрация данных),
- `onto-emitter` (публичная экспозиция профиля).
