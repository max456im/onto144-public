# Как внести вклад в onto144-public

Спасибо, что хотите участвовать в развитии онтологического реестра `onto144-public`!  
Этот документ описывает, как предложить новый **профиль синтетического разума** или **доменный лексикон**, а также какие требования необходимо соблюдать.

---

## Основные принципы

1. **Все материалы — под GPL-3.0-only.**  
   Это касается как кода, так и данных (профилей и лексиконов). Вы должны быть готовы открыть свои материалы под этой лицензией.

2. **Онтологическая целостность превыше всего.**  
   Профиль не может использовать термины, отсутствующие в соответствующем лексиконе. Лексикон не может содержать расплывчатых, двусмысленных или непроверяемых понятий.

3. **Этическая прозрачность.**  
   Каждый профиль должен явно указывать:
   - Свою этическую модель (например, `lec-2026`, `sub_duo-2026` или собственную, совместимую с *Three Laws of Ontogenesis*),
   - Область применения (медицина, право, гейминг и т.д.),
   - Источники терминов (только из соответствующего `.txt`-лексикона).

---

## Как предложить профиль

1. **Определите домен.**  
   Выберите существующий лексикон (`lexicons/`) или предложите новый (см. ниже).  
   Например: `medical.txt` → профиль помещается в `profiles/domain/medical/`.

2. **Создайте профиль в формате YAML.**  
   Используйте шаблон: [`examples/create_custom_profile_template.yaml`](../examples/create_custom_profile_template.yaml).  
   Обязательные поля описаны в [`docs/profile_anatomy.md`](../docs/profile_anatomy.md).

3. **Убедитесь в соответствии лексикону.**  
   Запустите проверку:
   ```bash
   python tools/check_profile_lexicon_compliance.py ваш_профиль.yaml lexicons/ваш_лексикон.txt
   ```

4. **Отправьте Pull Request в `profiles/community/pending/`.**  
   Имя файла: `ваше-имя-гггг.yaml` (например, `my-mind-2026.yaml`).

5. **Пройдите аудит.**  
   Сообщество и мейнтейнеры проверят:
   - Корректность структуры,
   - Соответствие лексикону,
   - Этическую нейтральность и безопасность,
   - Отсутствие дублирования.

   Только после успешного аудита профиль переместится в `profiles/community/verified/` и сможет использоваться в продакшене.

---

## Как предложить новый лексикон

1. **Обоснуйте необходимость.**  
   Новый домен (например, `biometrics.txt` или `zodiac_temperament.txt`) должен быть:
   - Чётко ограниченным по предметной области,
   - Основанным на авторитетных источниках (стандарты, законы, классификации),
   - Этически нейтральным или явно помеченным как специализированный (например, для гейминга).

2. **Создайте файл в `lexicons/`.**  
   Формат: одна лексема на строку, без дубликатов, без определений.  
   Подробнее: [`specs/lexicon-format.txt`](../specs/lexicon-format.txt).

3. **Предоставьте источники.**  
   В комментариях к PR укажите:
   - Источники терминов (ссылки на стандарты, документы, классификаторы),
   - Причины включения/исключения конкретных понятий.

4. **Отправьте Pull Request в `lexicons/`.**  
   После одобрения лексикон станет доступен для создания профилей.
   
Примечание: При предложении нового лексикона убедитесь, что он соответствует формату из specs/lexicon-format.txt. Pull request будет автоматически отклонён, если validate_lexicon.py завершится с ошибкой.

## Требования к оформлению

- Все файлы — в UTF-8.
- Никаких коммерческих маркеров, брендов или трейдмарков без явного разрешения.
- Запрещено включать:
  - Экстремистские, дискриминационные или дезинформирующие термины,
  - Личные данные реальных людей,
  - Неопределённые абстракции (типа «душа», «свобода» без контекста классификации).
